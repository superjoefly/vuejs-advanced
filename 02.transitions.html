Transitions

Overview

Vuejs is capable of applying transition effects when items are inserted, updated or removed from the DOM. Examples include:

  • automatically apply classes
  • use 3rd party CSS animation libraries
  • use JavaScript to manipulate the DOM
  • integrate 3rd party JavaScript animation libraries

Transitioning Single Elements/Components

A transition wrapper component will allow us to apply entering/leaving transitions to any element or component using the following contexts:

  • conditional rendering using v-if
  • conditional display using v-show
  • dynamic components
  • component root nodes

Hello There!

 

      <div id="demo1">

        <button @click="show = !show">Toggle</button>

        <transition name="fade">

          <p v-if="show">Hello There!</p>

        </transition>

      </div>

      -------------------------------

      var vm = new Vue({
        el: '#demo1',
        data: {
          show: true
        }
      })

      -------------------------------

      .fade-enter-active, .fade-leave-active {
        transition: opacity .5s;
      }

      .fade-enter, .fade-leave-to {
        opacity: 0;
      }

When elements wrapped in a transition component are inserted or removed, the following process occurs:

  1. If the target element has CSS transitions or animations applied, the transition classes will be appropriately added or removed.
  2. If the transition component has any JavaScript hooks, they will be called.
  3. If there are no CSS transitions/animations, and no JavaScript hooks are applied, the DOM operation for insertion and/or removal will be executed immediately on the next frame.

Transition Classes

The classes applied for enter/leave transitions are as follows:

  1. v-enter: starting state; added before element is inserted; removed one frame after element is inserted.
  2. v-enter-active: active state for enter; applied during the entire entering phase; added before element is inserted; removed when transition/animation finishes.
  3. v-enter-to: ending state for enter; added one frame after element is inserted; removed when transition/animation finishes.
  4. v-leave: starting state for leave; added immediately when leaving transition is triggered; removed after one frame.
  5. v-leave-active: active state for leave; applied during the entire leaving phase; added immediately when transition is triggered; removed when transition/animation finishes.
  6. v-leave-to: ending state for leave; added one frame after a leaving transition is triggered; removed when transition/animation finishes.

Each of these classes will be prefixed with the name of the transition. The v- prefix is default when a name is not provided.

Custom Transition Classes

We can specify custom transition classes by providing the following attributes:

  • enter-class
  • enter-active-class
  • enter-to-class
  • leave-class
  • leave-active-class
  • leave-to-class

These attributes will override the conventional class names. This is especially useful when combining Vue's transition system with existing CSS animation libraries:

Transition Durations

In most cases, Vue automatically figures out when a transition has finished. This may not always be desired. To specify an explicit transition duration (in milliseconds), we can use the duration prop on the transition component.

From the guide:

transition :duration="1000"

We can also specify separate values for enter and leave durations:

transition :duration="{ enter: 500, leave: 800 }"

JavaScript Hooks

We can also define JavaScript hooks in attributes:

I'm animated!

 

      <div id="demo6">

        <button @click="show = !show">Animate!</button>

        <transition

          @before-enter = 'beforeEnter'

          @enter = 'enter'

          @after-enter = 'afterEnter'

          @enter-cancelled = 'enterCancelled'

  

          @before-leave = 'beforeLeave'

          @leave = 'leave'

          @after-leave = 'afterLeave'

          @leave-cancelled = 'leaveCancelled'

          :css = 'false'

        >

          <p v-if="show">I'm animated!</p>

        </transition>

      </div>

      -------------------------------

      var vm = new Vue({
        el: '#demo6',
        data: {
          show: false,
          fontSize: 0,
          color: 'red'
        },
        methods: {
          // JavaScript Hooks
          beforeEnter(el) {
            console.log('beforeEnter!')
            el.style.color = this.color;
            this.fontSize = 0;
            el.style.fontSize = this.fontSize + 'px';
          },
          enter(el, done) {
            console.log('enter!')
            let size = 1;
            const interval = setInterval(() => {
              el.style.fontSize = (this.fontSize + size) + 'px';
              size++;
              if (size > 30) {
                clearInterval(interval);
                done();
              }
            }, 30)
          },
          afterEnter(el) {
            console.log('afterEnter!')
            el.style.color="green"
          },
          enterCancelled(el) {
            console.log('enterCancelled!')
          },
          beforeLeave(el) {
            console.log('beforeLeave!')
            el.style.color="yellow";
            this.fontSize = 30;
            el.style.fontSize = this.fontSize + 'px'
          },
          leave(el, done) {
            console.log('leave!')
            let size = 1;
            const interval = setInterval(() => {
              el.style.fontSize = (this.fontSize - size) + 'px';
              size++;
              if (size > 30) {
                clearInterval(interval);
                done();
              }
            }, 30)
          },
          afterLeave(el) {
            console.log('afterLeave!')
            el.style.fontSize = this.fontSize;
          },
          leaveCancelled(el) {
            console.log('leaveCancelled!')
          }
        }
      })

Note: when using JavaScript only transitions, the done callbacks are required for the enter and leave hooks. Otherwise they will be called synchronously and the transition will finish immediately.

Note: it is also a good idea to explicitly add v-bind:css="false" so that Vue can skip the CSS detection process. This will help to prevent accidental interference of CSS rules with the transition.

Transition on Initial Render

We can use the appear attribute to apply a transition on initial render of a node. By default this will use the transitions specified for entering and leaving, however we can also speicify custom CSS classes and custom JavaScript hooks:

This paragraph will transition (appear) when the page is loaded!

 

      <div id="demo8">

        <transition

          appear

          @before-appear="beforeAppear"

          @appear="appear"

          @after-appear="afterAppear"

          @after-appear-cancelled="appearCancelled"

        >

          <p>This paragraph will transition (appear) when the page is loaded!</p>

        </transition>

      </div>

      -------------------------------

      var vm = new Vue({
        el: '#demo8',
        data: {
          fontSize: 0
        },
        methods: {
          beforeAppear(el) {
            el.style.fontSize = this.fontSize + 'px';
            el.style.color = 'blue';
          },
          appear(el, done) {
            let size = 1;
            const interval = setInterval(() => {
              el.style.fontSize = (this.fontSize + size) + 'px';
              size++;
              if (size > 22) {
                clearInterval(interval);
                done();
              }
            }, 100)
          },
          afterAppear(el) {
            el.style.fontSize = '22px';
            el.style.color = 'red';
          },
          appearCancelled(el) {
            console.log('Cancelled!');
          }
        }
      })

The example above makes use of custom JavaScript Appear Hooks.

Transitioning Between Elements

We can transition between elements using v-if / v-on. Type some text into the input field and press enter to see the effect:

No items found...

 

      <div id="demo9">

        <input type="text" v-model="item" @keyup.enter="addItem" />

        <transition name="fade" mode="out-in">

          <ul v-if="items.length > 0" style="list-style-type: none;">

            <li v-for="item in items">

              <span style="color: red; font-size: 20px; position: relative; top: 2.5px; cursor: pointer;" @click="removeItem">&times;</span>

               {{ item }}

            </li>

          </ul>

          <p v-else>No items found...</p>

        </transition>

      </div>

      -------------------------------

      var vm = new Vue({
        el: '#demo9',
        data: {
          items: [],
          item: ''
        },
        methods: {
          addItem() {
            this.items.push(this.item);
            this.item = '';
          },
          removeItem(index) {
            this.items.pop(index);
          }
        }
      })

      --------------------------------

      .fade-enter-active, .fade-leave-active {
        transition: opacity .5s;
      }

      .fade-enter, .fade-leave-to {
        opacity: 0;
      }

Note: when transitioning between elements that have the same tag name, you must tell Vue that they are distinct elements by giving them unique key attributes.

Note: The default behavior of transitioning is for one element to transition out at the same time the other is transitioning in. This can lead to "jumping" around of the transitioned elements. In the example above, we use a transition mode of out-in so that the current element transitions out before the new element transitions in.

            var vm = this;
            return this.list.filter(function(item) {

            }, delay)
          }
        }
      })

        },

            this.show = true;
            this.stop = false;

          stopIt() {
            this.stop = true;
          },
          beforeEnter(el) {
            el.style.left = this.elementPosition + 'px';
          },
          enter(el, done) {
            let round = 1;
            const interval = setInterval(() => {
              el.style.left = (this.elementPosition + round * 10) + 'px';
              round++;
              if (round > 50) {
                clearInterval(interval);
                done();
                this.show = false;
              }
            }, this.speed)
          },
          leave(el, done) {
            let round = 1;
            const interval = setInterval(() => {
              this.elementPosition = 500;
              el.style.left = (this.elementPosition - round * 10) + 'px';
              round++;
              if (round > 50) {
                clearInterval(interval);
                done();
                vm.elementPosition = 1;
                if (!this.stop) {
                  this.show = true
                }
              }
            }, this.speed)
          }
        }
