Single File Components

Introduction

In many simple Vue projects, global components are defined using Vue.component, followed by new Vue({el: '#container'}) to target a container element i the body of every page.

This works well for small to medium sized projects, but in more complex projects, and when the entire frontend is driven by JavaScript, certain disadvantages become apparent:

  • Global definitions force unique names for each component
  • String templates lack syntax highlighting and require slashes for multiline HTML
  • CSS is not modularized into components
  • No build setup restricts us to HTML and ES5 JavaScript, rather than preprocessors like Pug and Babel

The above disadvantages are solved by using single-file components with a .vue extension, which is made possible with build tools such as Webpack or Browserify.

Following is an example of a .vue single-file component:


      // Hello.vue
        

      <template>

        <p>{{ greeting }} World!</p>

      </template>

        

      <script>

        export default {

          data() {

            return {

              greeting: 'Hello'

            }

          }

        }

      </script>

        

      <style scoped>

        p {

          font-size: 1.5em;

          text-align: center;

          color: orange

        }

      </style>

This provides for:

  • Complete syntax highlighting
  • CommonJS modules
  • Component-scoped CSS

Access to Pre-processors

Now, with single-file components, we have access to pre-processors such as Pug, Babel (with ES2015 modules), and Stylus for cleaner and more feature-rich components.


      // Hello Vue
        

      <template lang='jade'>

        div

          p {{ greeting }} World!

          other-component

      </template>

        

      <script>

        import OtherComponent from './OtherComponent.vue'

          

        export default {

          data() {

            return {

              greeting: 'Hello'

            }

          },

          components: {

            OtherComponent

          }

        }

      </script>

        

      <style lang="stylus" scoped>

        p

          font-size: 1.5em

          text-align: center

          color: orange

      </style>

The specific languages are just examples. If using Webpack with vue-loader, it also has first-class support for CSS Modules.

Separation of Concerns

Be aware that separation of concerns is not equal to separation of file types. Instead of dividing the code base into three huge layers that interweave, it makes more sense to divide them into loosely-coupled components. Inside a component, its template, logic and styles are inherently coupled making the component more cohesive and easier to maintain.

Even if we don't like the idea of single-file-components, we can still leverage the hot-reloading and pre-compilation features by separating our JavaScript into separate files:


      <!-- my-component.vue -->

      <template>

        <div>This will be pre-compiled</div>

      </template>

      

      <script src="./my-component.js"></script>

      <style src="./my-component.css"></style>

Module Build Systems in JavaScript

With .vue components, it is necessary to learn some additional tools such as Node Package Manager (NPM) and Modern JavaScript with ES2015/16 features.

A good starting point for new users is the webpack-simple template provided on Github. This template uses Webpack which is a module bundler that takes a number of "modules" and bundles them together in your final application.

For advanced users it is recommeded to browse github.com/vuejs-templates and select the template that's right for you. Just follow the instructions in the README file to generate a new project with vue-cli.
