Server Side Rendering

What is Server Side Rendering

Vuejs is a framework for building client-side applications. By default, Vue components produce and manipulate DOM in the browser as output. However, it is also possible to render the same components into HTML strings on the server, send them directly to the browser, and finally 'hydrate' the static markup into a fully interactive app on the client.

A server-rendered Vue.js app can also be considered 'isomorphic' or 'universal', in the sense that the majority of your app's code runs on both the server and the client.

Benefits of SSR

Server side rendering provides some benefits including:

  • Better SEO
  • Faster time-to-content

Drawbacks of SSR

There are also some trade-offs:

  • Development costraints
  • More involved build setup and deployment requirements
  • More server-side load

Before using SSR for you app you should ask whether or not you really need it, the deciding factor usually being the need for time-to-content performance gains. If time-to-content is critical, SSR can help achieve the best possible initial load performance.

For more information on SSR check out Vue's official SSR Guide.

Nuxt.js

To help simplify the task of properly configuring a production-ready server-rendered app, the Vuejs guide recommends Nuxt.js, a higher-level framework that is built on top of the Vue ecosystem.

Nuxt.js provies a streamlined development experience for writing universal Vue applications and can also be used as a static site generator with pages authored as single file components.
